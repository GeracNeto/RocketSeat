Tag de Entrada de Dados - input

Características

  • Um dos mais usados em formulários
  • Aceita os mais diversos tipos de dados
  • Um dos mais poderosos e complexos
  • Elevado número de combinações
  • Atributos mais utilizados:
    • type
    • name
    • id

Atributos

  • autocomplete

  • autofocus
    • Apenas um por página

  • disable

  • value

  • readonly
    • Igual o disabled porém não fica com cor opaca

  • form e name: serve para linkar o input para um formulário

  • require
    • Faz com que o preenchimento seja obrigatório

  • placeholder

Tipos de inputs

  • password
    • Exemplo de atributos:
      • minlenght / max lenght: mínimo e máximo de caracteres permitido pelo input
      • size: tamanho do campo input
      • pattern: expressão para validar os caracteres. Exemplo quando exige letra maiúscula etc...
      • inputmode: determina o tipo de entrada, se é só numérico, texto etc...

  • email
    • Exemplo de atributos:
      • multiple: permite colocar um ou mais emails por ","
      • pattern
      • list: linkado à uma datalist com id de mesmo nome para sugerir lista de emais emails

  • url
    • Exemplo de atributos:
      • placeholder
      • pattern: *\.com\.br/.* -> para ser obrigatório utilizar o .com.br
      • title: Caso no envio de erro ele mostra o title
      • list
      • spellcheck: habilita correção ortográfica para este input

  • File
    • Exemplo de atributos:
      • value: contém o arquivo a ser enviado
      • files: lista de um arquivo ou arquivos
      • accept: descreve que tipos de arquvios serão aceitos: .doc, .docx, .pdf, .audio/* (/* para mostrar que aceita todos os tipos de audio), etc...
      • multiple: permite o envio de múltiplos arquvios
      • Para formulários do tipo "file":
        • method="post"
        • enctype="multipart/form-data"

  • color
    • Exemplo de atributos:
      • value: para definir cor inicial
      • list: para pré determinar lista de cores (linkar com datalista) exemplo: datalist
